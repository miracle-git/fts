• 树是一种非线性的数据结构，表示一对多的关系(如文件目录结构，公司的组织架构，家族图谱等)
  • 与数组相比，数组具备快速通过下标访问，但如果根据元素内容查找或插入删除性能偏低
  • 与链表相比，链表虽然插入删除效率较高(中间位置仍然没那么高)，但是查找需要从头(或从尾)开始
  • 与哈希表相比，哈希表虽然查询，插入和删除效率都很高，但空间利用率不高(有很多空位)，且元素之间时无序的导致难以按固定顺序遍历，也难以快速找到最大最小值等
  • 树可以解决数组，链表，哈希表的部分缺点(查询，插入和删除方便，空间利用率高)，但实现难度较数组，链表，哈希表都稍复杂
  • 二叉树：树中每个节点最多包含两个子节点(可只包含左子节点或右子节点)，二叉树可为空
  • 二叉树包含完美二叉树(或满二叉树，除叶子节点之外，所有节点都有两个子节点)和完全二叉树(除最后一层节点外，其他各层都达到最大节点数，最后一层从左到右依次连续存在，可缺少最右侧节点)
  • 二叉搜索树(BST)：满足以下3条件，左子树上所有节点键值小于根节点(或祖先节点)的键值，右子树上所有节点键值大于根节点(或祖先节点)的键值，左右子树均仍然是二叉搜索树
