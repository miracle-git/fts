• 图是包含顶点(Vertex)和边(Edge)以及之间关系的数据结构(树是一种特殊的图，如:人与人之间关系网，地铁线路图)
  • 图由一组顶点组成(Vertexes)
  • 图由一组边组成(Edges), 边表示顶点和顶点之间的连线, 边可以是无向的(A--B)，也可以是有向的(A-->B)
  • 两个靠边相连的顶点叫相邻顶点(如: 0和1, 1和2都是相邻点)，一个顶点对应相邻顶点的数目叫做度(0和1顶点对应度分别为2和4)
  • 路径是由一系列顶点v1,v2,...,vn组成的连续序列(如图中0->1->5->9，0->3->6->8->9)，而简单路径是由不重复的顶点构成(如图中0->1->5->9)，回路是起点和终点一致的路径(如图中的0->1->5->6->3->0)
  • 无向图是指任意两个顶点之间都是没有方向的(A--B)，有向图是指顶点之间都是有方向的(A-->B不代表B-->A)
  • 无权图是指任意顶点之间不带权重的(即无法对比多条路径的"孰优孰劣")，有权图是指任意顶点之间带有权重(权重可代表距离，时间或价位等，因此可以"优中挑优")
  • 图可由邻接矩阵(Adjacency Matrix)来表示，每个顶点和一个整数(数组下标)相关联，而顶点和顶点之间是否有边用0或1(权重)来占位(但存在大量无边的情况，对空间存储来说比较浪费)
  • 图可由邻接表(Adjacency Table)来表示，每个顶点和多个相邻点列表(可用数组/链表/哈希表)来存储(邻接表顶点"出度"非常容易找到，但"入度"却比较麻烦)
  • 图的遍历需要将每个顶点都要访问且不能重复访问，为了记录是否被访问过，使用白色(White)表示从未被访问过，使用灰色(Gray)表示被访问过但未被探索过，使用黑色(Black)表示被访问过且被探索过，不管是BFS还是DFS，都需要明确指定第一个访问的顶点
  • 广度优先遍历(BFS, Breadth-First Search): 基于队列实现，先入队列的顶点优先访问，当出队时其临接顶点依次入队，直到所有顶点全部出队为止
  • 深度优先遍历(DFS, Depth-First Search)：基于栈或递归实现，将顶点存入栈中，并沿着路径去不断探索，当存在新的邻接顶点则一直访问，直到不存在为止
